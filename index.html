ML Pipeline

End-to-end walkthrough of how the model was built from raw data to deployment.

1
Data ingestion
5,000 political social media posts from Twitter and Facebook. Crowd-labeled via CrowdFlower with bias (neutral/partisan), audience, and message type annotations. Loaded from CSV with latin1 encoding.
5k posts2 platforms
2
Text cleaning
URLs, @mentions, and # symbols removed with regex. Text lowercased, extra whitespace collapsed. Hashtag text content preserved to retain political keywords.
regexlowercase
3
Oversampling for class balance
Raw data is 73.8% neutral. Minority partisan class is oversampled with replacement (sklearn resample) to match the majority class size — producing a 50/50 training distribution of 3,689 × 2.
oversampling3,689 × 3,689
4
TF-IDF vectorization
15,000 top features, unigrams + bigrams, sublinear TF scaling. Captures both single keywords and common partisan/neutral two-word phrases (e.g. "working families", "vote against").
TF-IDF15k featuresbigrams
5
LightGBM classifier
400 estimators, learning rate 0.05, 63 leaves, balanced class weights. Fast gradient boosting on the sparse TF-IDF matrix. Trained on 80% of balanced data, evaluated on stratified 20% hold-out.
LightGBM400 trees
6
Flask API + web interface
Model served via Flask /predict endpoint. Frontend calls it via fetch() and renders verdict, confidence bars, and top contributing n-grams in real time.
FlaskREST APIlive inference
